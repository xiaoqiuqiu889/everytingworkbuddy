02每个工具做什么

说需求（或报身份「我是做 XX 的」），入口 efc 会按语义自动路由到对的子技能，你不必记名字。

/efc
统一入口：实质任务的第一动作，自动路由到下面 11 个子技能
/efc-plan-feature
先规划——把需求拆成文件级、带验收标准的实现计划
/efc-tdd-workflow
测试驱动开发 RED→GREEN→REFACTOR，覆盖 80%+
/efc-code-review
代码审查，按 Blocker/Should/Nice 分级给修复
/efc-build-fix
从根因修构建/编译错误，一次一个再重验
/efc-refactor-clean
重构与死代码清理，行为不变、小步验证
/efc-security-review
安全审查，按风险找漏洞与密钥泄露
/efc-verify
交付前逐条验收闸门——没跑不许说「做完了」
/efc-checkpoint
存进度快照，下次直接续上不丢上下文
/efc-learn
从会话提炼可复用套路，写入长期记忆
/efc-github-push
沙箱/无头环境非交互推送 GitHub，token 不落盘
/efc-profile
按身份检索、推荐能力组合并写入个人档案

·灵魂：四步编排外壳

EFCC 不是「另一条路」，而是套在所有实质任务外面的外壳。中间「干活」交给谁都行，但收尾的验收和沉淀强制、不豁免。

STEP 1
① 计划
拆目标、交付物、分步与验收标准
→
STEP 2
② 干活
交给对口领域技能 / 原生工具
→
STEP 3 · 硬闸门
③ 验收
对照标准逐条勾选，产出可核验清单
→
STEP 4
④ 沉淀
可复用套路写进记忆或存成新技能

📌 什么算「实质任务」（满足任一即触发外壳）：① 产出文件；② 需 ≥3 步；③ 预计 ≥15 次工具调用。纯问答/闲聊不套。

03我做了哪些事，保证「真能调用到工具」

两款助手扩展机制完全不同——Claude 靠自动加载的技能，Codex 靠手动斜杠命令。要「同一套」在两端都真能触发，我做了这几件事：

① 按各自机制精确落位

相关时自动加载 / /name		~/.codex/prompts/<name>.md
斜杠命令 /name（自动剥 frontmatter）
常驻纪律~/.claude/CLAUDE.md
每会话自动注入		~/.codex/AGENTS.md
每会话自动注入
MCP 连接器~/.claude.json（JSON）~/.codex/config.toml（TOML）
子代理~/.claude/agents/*.md~/.codex/prompts/agent-*.md

② 用「触发纪律」对抗漏用，并逐条实测

✓
常驻记忆写入「0 号动作」

把「遇实质任务，第一个动作先加载 efc」写进两端每会话自动注入的记忆文件，并附「反暗合」表——用待办清单拆任务 ≠ 走外壳、自写脚本 ≠ efc-verify。

✓
独立子代理真实触发验证

派了无本会话上下文的独立子代理跑真实任务，观察它是否真的用 Skill 工具，而非「嘴上说走了四步」。

结果：子代理真实调用 Skill 3 次（efc → efc-plan-feature → efc-verify），跑了 10 项验收，产出文件经独立运行与 wc 结果一致 ✓
✓
幂等 & 不破坏已有配置

重复安装：记忆块用 sentinel 整体替换（不堆叠），MCP 已存在则跳过——实测未覆盖用户原有的 node_repl 服务。

CLAUDE.md / AGENTS.md 各仅 1 个规则块 · config.toml 无重复 · node_repl 完好
✓
Clean-machine + 自检脚本

用临时 HOME 模拟全新机器从零安装通过；doctor.mjs 校验两端组件就位、源与已装一致（防漂移）。

12 技能 · 21 Codex 命令 · 双端规则块各 1 · doctor 全绿 ✓
✓
能力检索器按 GitHub 生态热度加权排序

match.mjs 同时扫描 ~/.claude/skills 与 ~/.codex/prompts 两种布局，按身份自述打分推荐，并标记「现在就能用 / 需安装」。热度信号 = 仓库 star + 技能名在 issue/PR 提及数；EFCC 原生固定最高分。

实测："做 PPT" 查询下热度开启时 top1 为 pptx（Anthropic 官方），关闭热度后 pptx 掉出 top5 —— 证明热度确实影响排序 ✓

04怎么用（应用方法）

一条命令装两端，随后说需求即可触发。

# 安装（默认两端都装） node EFCC/scripts/install.mjs # 只装一端 node EFCC/scripts/install.mjs --target claude node EFCC/scripts/install.mjs --target codex # 自检 / 检索能力 node EFCC/scripts/doctor.mjs node EFCC/scripts/match.mjs "我是游戏策划"
CLAUDE 说需求即自动路由，或显式： /efc 帮我实现登录功能 /efc-verify 验收一下 报身份 → 自动走 efc-profile 「我是做数值系统的游戏策划」
CODEX 用斜杠命令显式触发： /efc /efc-plan-feature /efc-verify （prompts 是手动命令，四步 常驻性靠 AGENTS.md 保证）

新项目启用：把 examples/CLAUDE.md（Claude）或 examples/AGENTS.md（Codex）复制到项目根——每会话注入、子代理也读。
可拓展：在 user/ 覆盖层加技能/准则/MCP，重跑安装即生效且自动并入检索，重装不丢。
