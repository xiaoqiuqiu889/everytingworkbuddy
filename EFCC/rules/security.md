# Rule: Security — 安全准则

## 红线（不可违反）

1. **绝不硬编码密钥**：API key、密码、token、连接串一律走环境变量 / 密钥管理服务。代码、日志、注释、提交里都不出现真实凭据。
2. **`.env` 不进版本库**：`.env`、凭据文件加入 `.gitignore`；提供 `.env.example` 占位。
3. **外部输入零信任**：所有来自用户/网络/文件/第三方的数据默认不可信，先校验再使用。

## 强制检查

- **注入防护**：参数化查询（防 SQL 注入）；避免拼接执行命令；输出转义（防 XSS）；限制路径（防穿越）。
- **认证授权**：权限校验放服务端，不信任前端；最小权限原则；会话/token 安全存储与过期。
- **传输与存储**：对外传输用 HTTPS/TLS；敏感数据静态加密；PII 按合规处理。
- **依赖**：不用已知漏洞版本；定期审计依赖。
- **默认安全**：生产关闭调试模式；错误信息不泄露栈/内部结构；CORS 最小化。

## 提交前自查

- [ ] 无硬编码密钥
- [ ] 外部输入已校验
- [ ] 无危险的动态执行（eval / 拼接命令）
- [ ] 敏感数据未打印到日志
