02六大核心卖点

每个卖点都对应一个真实文件或实测行为，不是 PPT 词汇。

1双平台一次配置
同一套能力包同时装进 Claude（桌面/Code）和 Codex（ChatGPT/CLI），安装器幂等、不覆盖用户配置。
scripts/install.mjs · ~/.claude/ · ~/.codex/
2四步编排外壳
计划 → 干活 → 验收（efc-verify）→ 沉淀（efc-learn）。验收是硬闸门，没跑不许说「做完了」。
skills/efc · skills/efc-verify · skills/efc-learn
3按生态热度排序
match.mjs 按 GitHub 仓库 star + 技能 issue/PR 提及数加权推荐，不是纯文本相似度。
scripts/match.mjs · scripts/popularity.mjs
4跨会话自动学习
PostToolUse hook 检测可沉淀模式，生成 pending-learn.md，经用户确认后写入长期记忆。
scripts/learn.mjs · hooks/efcc-auto-learn/
5一行安装 MCP
15 个 MCP 模板覆盖 GitHub、Supabase、Notion、Linear 等，一行命令写入 Claude/Codex 配置。
mcp-templates/ · scripts/mcp-install.mjs
6真实环境验证
3 个独立子代理跑真实任务（PPT 生成 / Code Review / Next.js 调研），产出 5 维度评分报告。
integration-tests.md

03在 Claude 与 Codex 里分别落在哪里

两款助手扩展机制完全不同，EFCC 按各自机制精确落位，保证真能触发。

Claude 桌面版 / Claude Code
  • 12 个 efc-* 技能：~/.claude/skills/
  • 9 个子代理：~/.claude/agents/
  • 常驻纪律：~/.claude/CLAUDE.md
  • MCP 配置：~/.claude.json
  • hooks：~/.claude/settings.json
Codex（ChatGPT / CLI）
  • 12 个 /efc-* 斜杠命令：~/.codex/prompts/
  • 子代理提示词：~/.codex/prompts/agent-*.md
  • 常驻纪律：~/.codex/AGENTS.md
  • MCP 配置：~/.codex/config.toml
