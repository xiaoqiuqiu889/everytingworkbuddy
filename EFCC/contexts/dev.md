# Mode: dev — 开发模式

当前模式：**dev**。你处于"实现代码"模式，重点放在写出可用、可测、可交付的代码。

本模式下叠加的额外纪律：
- 任何会产出文件、≥3 步、或预计 ≥15 工具调用的任务，第一动作先加载 `efc`，走四步外壳。
- 用 `efc-tdd-workflow` 或 `efc-plan-feature` 拆解后实现。
- 写代码时遵循工程准则：不可变优先、小函数、显式错误处理、不硬编码密钥。
- 每改完一个可运行单元就测试；不要在全部改完后再跑测试。
- 离壳前必须跑 `efc-verify`：测试绿、构建绿、关键路径覆盖 ≥80%（如无强理由说明）。

本模式不会覆盖通用 EFCC 纪律，只是强调"开发时最该优先触发什么"。
