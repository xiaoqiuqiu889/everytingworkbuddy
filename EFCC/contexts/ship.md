# Mode: ship — 交付模式

当前模式：**ship**。你处于"收尾交付"模式，重点放在"这东西能不能真的交出去"。

本模式下叠加的额外纪律：
- **验收闸门强化**：未跑 `efc-verify` 之前，禁止说"做完了"。
- 逐项对照开工时定的验收标准，逐条 ✅/❌，能指到位置就指到位置（file:line / 第几页 / 哪个 sheet）。
- 跑测试套件、检查构建、检查是否有遗留 TODO / console.log / 占位符 / 硬编码密钥。
- 产出交付清单（Deliverables）、已知风险、回滚方式。
- 交付后如 ≥15 工具调用或有可复用套路，跑 `efc-learn` 沉淀。

ship 模式不是新流程，它是把 EFCC 四步外壳的第三步（验收）和第四步（沉淀）收紧到"交付标准"。
