# AGENTS.md — EFW 编排外壳（项目级，子 agent 也读）

> **为什么有这个文件**：用户级 `~/.workbuddy/MEMORY.md` 里的 EFW 纪律**只对主对话生效**——子 agent / 子代理是独立上下文，**不加载用户级 MEMORY**。而项目根的 `AGENTS.md`（或 `CODEBUDDY.md`）会被**每个会话开头全量注入**，是让 EFW 触达"所有在本项目干活的 agent（含子 agent）"的唯一可靠通道。
>
> **用法**：把本文件复制到你项目根目录（命名 `AGENTS.md`；若项目已有 `CODEBUDDY.md`，把本段并进去即可——CODEBUDDY.md 优先级更高）。删改成你项目的实际约定。

---

## 🍶 本项目采用 EFW 编排外壳

**EFW 是套在所有实质任务外面的外壳，不是可选的另一条路。** 无论你是主对话还是被派来的子 agent，接到实质任务都走这四步：

```
① 计划  →  ② 干活  →  ③ 验收  →  ④ 沉淀
          ↑ 交给对口领域技能或原生工具
```

**什么算"实质任务"——按「任务意图指向什么」判，不按「本次动作是否产文件」判**：① 请求**指向交付物**（报告/PPT/网页/代码/设计稿…，含 .xlsx/.pptx/.docx/.pdf/.html），**哪怕当下只要建议也进"预备态"**——可以先给建议但须声明"此为咨询，一旦转产出即加载 efw"，别用"这步只是建议"拖延；或 ② 需 ≥3 步；或 ③ 预计 ≥15 次工具调用。
→ 纯问答 / 查一条资料 / 闲聊 / 单步文本（**不指向任何交付物**），**不算**，直接做，别套外壳（套了是负担）。

> 🔁 **意图升级重触发**：命中「做成/导出/生成/落地/帮我做一份/直接出/转成文件/写个…」→ 无论之前是否按咨询处理，**立刻回到 0 号动作加载 efw**。
> 🛡️ **高危类目硬默认**：PPT / 设计 / 文档 / 数据分析 / 报告 默认视为"指向交付物"，直接起壳，不先问"是否实质"。
> 🪶 **lite 轻量模式**：纯咨询且指向交付物时，只做 ①一句话计划 + ③预置验收标准，不必跑全套。

> ⛔ **0 号动作**：判定为实质任务 → **第一个动作先加载 `efw` 技能**，先于任何领域工具。EFW 载体是可选 skill、系统不会自动加载，必须你显式发起——这是机械默认动作，不等被点名。
> 🚫 **反"暗合"**：TaskCreate 拆任务 ≠ 走外壳；自写校验脚本 ≠ efw-verify；写 memory 笔记 ≠ efw-learn；"心里过了一遍" ≠ 真走技能。**神似≠触发。**

> 🚦 **完成定义（DoD，硬闸门）**：实质任务只有跑完 ③验收 才算"完成"——**没做逐条验收前，不许说"做完了"**。文档/数据/设计任务（pdf/xlsx/pptx/docx）最容易被当成"纯领域技能任务"而跳过闸门，切勿豁免。
> 🔗 用 TaskCreate 拆任务时，**最后一项固定加"验收 + 沉淀"**，让闸门变成清单里绕不过的一项；每项任务即一条验收标准。

**四步怎么走：**
1. **计划**：先想清楚目标、交付物长什么样、分几步、每步验收标准。
   - 已加载的对口领域技能（如设计/文档/数据/研究类技能）自带 plan/大纲的，**视为满足本步**，不必再叠 `efw-plan-feature`（避免双规划）。
   - 只有"纯规划/方案/拆解"任务、或无对口技能时，才显式用 `efw-plan-feature` / `agent-planner`。
2. **干活**：交给最对口的领域技能或原生工具。EFW 不跟它抢。
3. **验收（强制闸门，领域技能不豁免）**：产出后对照第①步的验收标准**逐条勾选、给出可核验的验收清单**，而不是肉眼看看或临时抽查就交差。
4. **沉淀**：≥15 工具调用或发现可复用套路的任务，收尾把经验/流程沉淀下来（可复用流程直接存成新 skill）。

> ⚠️ 最常见的漏用：遇到一个"很能干"的领域技能，就把它当成 EFW 的替代品、把③验收④沉淀整个丢了。**记住：领域技能负责"干活"，EFW 外壳负责"验收+沉淀"，两者并存。**

## 本项目启用的具体能力（按需删改）

```yaml
# 举例，换成你项目的真实约定
skills:
  - efw-plan-feature   # 规划/方案/拆解（产品/设计/运营/工程方案通用）
  - efw-verify         # 交付前逐条验收
  - efw-learn          # 收尾沉淀
connectors:            # 连接器面板授权，本项目 < 10 个
  - github
rules:                 # 写代码时额外遵循（见 EFW rules/）
  - 安全 / 编码 / 测试 / Git
```
